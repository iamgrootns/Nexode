Embora o nexode possa rodar direto no Windows, recomendamos usar [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) para a melhor experiência. O WSL oferece um ambiente Linux que funciona de forma integrada com os recursos do nexode.

:::tip[Por que WSL?]
O WSL oferece melhor desempenho de sistema de arquivos, suporte completo a terminal e compatibilidade com as ferramentas de desenvolvimento das quais o nexode depende.
:::

---

## Configuração

<Steps>

1. **Instale o WSL**

   Se ainda não instalou, [instale o WSL](https://learn.microsoft.com/en-us/windows/wsl/install) usando o guia oficial da Microsoft.

2. **Instale o nexode no WSL**

   Depois de configurar o WSL, abra o terminal do WSL e instale o nexode usando um dos [métodos de instalação](/docs/).

   ```bash
   curl -fsSL https://nexode.ai/install | bash
   ```

3. **Use o nexode pelo WSL**

   Vá para o diretório do seu projeto (acesse arquivos do Windows via `/mnt/c/`, `/mnt/d/` etc.) e execute o nexode.

   ```bash
   cd /mnt/c/Users/YourName/project
   nexode
   ```

</Steps>

## App desktop + servidor WSL

Se você prefere usar o app desktop do nexode, mas quer rodar o servidor no WSL:

1. **Inicie o servidor no WSL** com `--hostname 0.0.0.0` para permitir conexões externas:

   ```bash
   nexode serve --hostname 0.0.0.0 --port 4096
   ```

2. **Conecte o app desktop** em `http://localhost:4096`

:::note
Se `localhost` não funcionar no seu ambiente, conecte usando o IP do WSL (no WSL: `hostname -I`) e use `http://<wsl-ip>:4096`.
:::

:::caution
Ao usar `--hostname 0.0.0.0`, defina `NEXODE_SERVER_PASSWORD` para proteger o servidor.

```bash
NEXODE_SERVER_PASSWORD=your-password nexode serve --hostname 0.0.0.0
```

:::

## Cliente web + WSL

Para a melhor experiência web no Windows:

1. **Execute `nexode web` no terminal WSL** em vez do PowerShell:

   ```bash
   nexode web --hostname 0.0.0.0
   ```

2. **Acesse pelo navegador do Windows** em `http://localhost:<port>` (o nexode mostra a URL)

Executar `nexode web` a partir do WSL garante acesso correto ao sistema de arquivos e integração com o terminal, continuando acessível no navegador do Windows.

---

## Acessar arquivos do Windows

O WSL pode acessar todos os arquivos do Windows pelo diretório `/mnt/`:

- Unidade `C:` → `/mnt/c/`
- Unidade `D:` → `/mnt/d/`
- E assim por diante...

Exemplo:

```bash
cd /mnt/c/Users/YourName/Documents/project
nexode
```

:::tip
Para uma experiência mais fluida, considere clonar/copiar seu repositório para o sistema de arquivos do WSL (por exemplo em `~/code/`) e executar o nexode por lá.
:::

---

## Dicas

- Mantenha o nexode rodando no WSL para projetos armazenados em unidades do Windows - o acesso aos arquivos fica fluido
- Use a [extensão WSL do VS Code](https://code.visualstudio.com/docs/remote/wsl) junto com o nexode para um fluxo de desenvolvimento integrado
- Sua configuração e suas sessões do nexode ficam armazenadas no ambiente WSL em `~/.local/share/nexode/`
